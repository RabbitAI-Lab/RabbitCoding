- 采用“容器-展示”分离模式，复杂页面（如 ContentArea）负责业务逻辑与状态协调，子组件（如 AgentChat, RightPanel）仅负责 UI 渲染与事件回调。
- 所有与后端异步通信的 Tauri invoke 调用均包裹在 try-catch 中，并通过状态更新（如 status: 'error'）在 UI 层反馈错误，避免未捕获异常导致界面崩溃。
- 使用自定义 Hook（如 useAgent, useWorkspaces, useLocalStorage）封装副作用与状态逻辑，确保组件内部逻辑简洁且可复用。
- 流式消息处理采用“增量追加”策略，通过识别 message subtype（如 text_delta, thinking_delta）实时更新 UI，而非等待完整消息返回。

# 移除对话统计行并改用逐轮 Markdown 复制

_来源：0aa6df7 → dc1bf71 提交周期内记录的编码计划——内容为规划时意图，实现可能滞后或有出入。_

**状态：** accepted

## 背景
用户对 AI 回复末尾显示的“任务完成”统计行（包含时长、价格、Token 汇总）反馈不佳，认为其缺乏实用价值且干扰阅读。同时，用户放弃了此前提出的“逐轮显示 Token 明细”的需求，转而希望更便捷地获取纯文本内容。

## 决策驱动
- 界面简洁性
- 用户实用性
- 需求变更

## 备选方案
- **优化统计行显示（分项显示 Token/缓存/价格）** _（已否决）_ — 优点：信息透明度高；缺点：占用垂直空间；用户明确表示该信息无价值；增加了前端渲染复杂度
- **完全移除统计信息且不替代** _（已否决）_ — 优点：界面最简洁；缺点：用户失去了快速获取纯文本内容的便捷途径
- **移除统计行，替换为“复制 Markdown”按钮** — 优点：消除视觉干扰；提供高价值的文本提取功能；符合开发者用户习惯；缺点：需要新增 CopyMarkdownButton 组件及剪贴板逻辑

## 决策
从 `AgentMessage.tsx` 中彻底移除成功状态下的统计行（时长、价格、Token）。在每轮 AI 回复末尾引入 `CopyMarkdownButton` 组件，允许用户一键复制该轮次的所有原始 Markdown 内容。错误状态仅保留错误文本提示。

## 影响
对话界面更加清爽，聚焦于内容本身。用户获得了更实用的内容导出能力。此前为支持逐轮 Token 显示而进行的协议层改动（turnUsages）在此决策中被废弃或不再用于 UI 展示。

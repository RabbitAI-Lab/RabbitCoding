- **入口与生命周期**：`src/main.rs` 仅作为二进制入口调用 `lib.rs::run()`；`lib.rs` 通过 `tauri::Builder` 组装插件（`pty`, `fs`, `window-state` 等）并注册全局状态（`Database`, `SidecarState`）。
- **模块化领域层**：`src/` 下按功能垂直切分为多个模块：`sidecar.rs`（AI Agent 子进程管理）、`db.rs`（SQLite 数据持久化）、`auth.rs`（Casdoor OAuth 回调服务）、`integration.rs`（GitHub Device Flow）、`gitnexus.rs`（代码索引工具链管理）、`ecc.rs`（Claude 扩展管理）以及 `network.rs`（网络诊断）。
- **通信边界**：所有功能均通过 `#[tauri::command]` 暴露为 IPC 接口，前端通过异步调用触发后端逻辑；复杂任务（如安装、索引、网络探测）在 `tokio::task::spawn_blocking` 或独立线程中执行，并通过 Tauri Events (`emit`) 向前端推送实时进度。
- **资源与环境隔离**：生产环境下通过 `lib.rs` 注入内置 Node.js 运行时路径至 `PATH`，并使用应用私有目录（`app_data_dir`）作为 npm 全局前缀，确保 CLI 工具（如 `gitnexus`）的安装与执行不依赖系统环境且具备权限安全性。

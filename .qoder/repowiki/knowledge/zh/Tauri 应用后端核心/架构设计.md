- **入口与配置**：`src/lib.rs` 为应用入口，通过 `tauri::Builder` 注册插件（fs, dialog, shell, notification 等）并挂载全局状态（`SidecarState`, `wiki::WikiQueueState`）。`tauri.conf.json` 定义主窗口与透明宠物窗口，并声明 `resources/sidecar` 和 `resources/node-runtime` 为打包资源。
- **模块分层**：
  - **Wiki 引擎** (`src/wiki/`)：纯 Rust 实现的 AI 驱动文档生成器。包含 `generator`（编排 repo/workspace 两级生成）、`ai`（Anthropic API tool_use 循环）、`tools`（文件读写工具）、`queue`（全局串行任务队列）及 `commands`（Tauri 命令暴露）。
  - **Sidecar 桥接** (`src/sidecar.rs`)：管理 Node.js Sidecar 子进程的生命周期。在启动时注入隔离的环境变量（如 `CLAUDE_CONFIG_DIR`），并通过 stdin/stdout 实现双向 JSONL 通信。
  - **系统交互**：集成 `rusqlite` 进行本地持久化，通过 FFI 实现 macOS/Windows 下的鼠标穿透检测（宠物窗口）及应用激活，并提供 Git 信息获取、通知发送等原生能力。
- **依赖方向**：前端通过 Tauri IPC 调用后端命令；后端通过 `reqwest` 调用外部 AI API，通过 `std::process` 管理 Sidecar 子进程。

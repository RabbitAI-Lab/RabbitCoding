# 模型选择器采用最新模型与自定义模型双Tab布局

_来源：31586ac → 7909aa7 提交周期内记录的编码计划——内容为规划时意图，实现可能滞后或有出入。_

**状态：** accepted

## 背景
引入线上模型后，模型列表来源变得复杂（动态线上列表 + 静态本地列表），需要在有限的 UI 空间内清晰展示并允许用户快速切换。

## 决策驱动
- UI 清晰度与分类管理
- 突出线上最新模型的可用性
- 保持自定义模型管理的原有习惯

## 备选方案
- **双 Tab 分组展示（最新模型/自定义模型）** — 优点：逻辑隔离清晰，用户可明确感知模型来源；便于对线上模型进行特殊状态标记（如“在线”、加载中）；缺点：增加了一次点击操作（需切换 Tab）
- **单一列表混合排序** _（已否决）_ — 优点：操作步骤最少；缺点：线上模型与本地模型混杂，难以区分来源；无法有效展示线上模型的加载状态或空状态

## 决策
改造 `src/components/common/ModelSelector.tsx`，在 Popover 顶部增加 Tab 切换。'最新模型' Tab 展示从 Portal 获取的在线模型，'自定义模型' Tab 保留原有本地配置列表。选中线上模型时 ID 携带 `__online__:` 前缀。

## 影响
ModelSelector 组件复杂度增加，需处理两个数据源的 loading 和 empty 状态；需新增对应的 i18n 文案（如 `latestModels`, `onlineTag` 等）。
